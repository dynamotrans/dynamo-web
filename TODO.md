## рҹҸ—пёҸ Stack y decisiones para el backend (sesiГіn 2026-06-20)

Bloque consolidado para tener todo en un sitio cuando arranquemos la implementaciГіn. Plan estimado: **3-4 semanas concentradas para MVP** sin Holded ni n8n; **5-7 semanas** si aГұadimos los dos.

### Plataforma
- **Supabase** (Postgres gestionado + Auth + Storage + Edge Functions + Realtime + RLS). Cuenta y proyecto pendientes de crear.
- **Vercel** ya conectado al repo. Se mantiene para hosting del front (web pГәblica + futuro panel Next.js). Cuando el panel pase a producciГіn real, activar Deployment Protection para previews.
- **Dominio panel**: `panel.dynamotrans.com` (subdominio dedicado para el portal). Apuntar al deploy de Vercel cuando el back estГ© listo.

### Email transaccional вҶ’ **Postmark** (decidido 2026-06-20)
- RazГіn: deliverability lГӯder del mercado, separa estricto transaccional vs marketing. En B2B logГӯstico con facturas y albaranes legales no se puede permitir que un email "se pierda" en spam.
- Coste: **$15/mes hasta 10k emails** (asumible desde el dГӯa 1).
- Se descartГі Resend aunque tiene free tier 3k/mes вҖ” la diferencia anual con Postmark es marginal y el upside en deliverability + soporte vale la pena.
- Free tier de Postmark (100/mes) basta para fase de pruebas, luego se pasa al plan de pago.
- Setup: cuenta en `postmarkapp.com`, aГұadir dominio `dynamotrans.com` con 3 registros DNS (SPF / DKIM / Return-Path), generar API token, guardarlo en `POSTMARK_API_KEY` de Supabase Edge Functions. Llamadas desde Edge Functions con SDK npm.
- Casos de uso: confirmaciГіn de carga creada, cambio de estado de carga (programada вҶ’ en ruta вҶ’ entregada), albarГЎn disponible, recuperaciГіn de contraseГұa, alertas de incidencia al equipo, recordatorios.

### WhatsApp transaccional вҶ’ **WhatsApp Cloud API (Meta/Facebook)** (ya provisionado por el cliente)
- El cliente confirmГі que **ya tiene la API de WhatsApp dada de alta con Facebook + nГәmero de telГ©fono verificado**. Pendiente solo de cablear cuando entre el backend.
- Casos de uso: avisos al cliente cuando se confirma transportista, recordatorio de carga el dГӯa anterior, alerta de incidencia, recuperar contraseГұa si email no llega.
- Cliente puede optar por canal preferido (email / whatsapp / ambos) por tipo de evento.
- Coste de Meta: gratis hasta 1.000 conversaciones iniciadas por usuario/mes; ~0,03-0,06 вӮ¬ por conversaciГіn adicional segГәn paГӯs (EspaГұa es barato).
- ImplementaciГіn: Edge Function que hace POST a `graph.facebook.com/v18.0/{phone-number-id}/messages` con token de Meta en `META_WA_TOKEN`. Plantillas aprobadas en Meta Business Manager.

### Naming convention de columnas/campos
- **`snake_case` en espaГұol** (confirmado 2026-06-20 leyendo la base del cliente en Google Sheets `IA-DYNAMO-2030`, ID `11hUKWFHu0cirk-IAGIiVPETFxwt5_Lo4QAHmYlUtO8o`, propietario `info@dynamotrans.com`).
- Los nombres usados en el Sheets (`referencia_continua_carga`, `fecha_hora_asignacion_camion`, `transportista_concatenar`, etc.) son la **fuente de verdad** para el schema de Supabase y para los campos del JSON en frontend.
- TODO pendiente: refactor del mockup actual para renombrar `fechaAsignada` вҶ’ `fecha_hora_asignacion_camion`, `transportistaNombre` вҶ’ `transportista_concatenar`, `matricula` вҶ’ `matriculas_autorizadas`, etc. Sale gratis con un find&replace masivo cuando se haga la primera sincronizaciГіn.

### MigraciГіn AppScript вҶ’ Edge Functions
- La lГіgica de negocio que el cliente ya tiene en AppScript del Sheets (tarifador, validaciones, cГЎlculos de margen, etc.) se porta a Supabase Edge Functions.
- Esfuerzo estimado: 1-2 semanas para alguien que ya maneja JS (el cliente lo maneja). La lГіgica se reaprovecha en un 95%, solo cambia el plumbing de I/O (SpreadsheetApp.getRange вҶ’ supabase.from().select()).
- Triggers temporales de AppScript (`onEdit`, `onChange`, `time-based`) вҶ’ Postgres triggers o `pg_cron`.

### Plan por fases (3-4 semanas MVP sin Holded ni n8n)
- **Semana 1** вҖ” Schema + RLS + Auth + conectar dashboard: cliente entra y ve sus cargas reales de Supabase.
- **Semana 2** вҖ” Tarifador en Edge Function + POST de cargas/presupuestos + estados dinГЎmicos.
- **Semana 3** вҖ” Editor de cargas con flujo legal + incidencias + Storage para fotos + pulido del frontend.
- **Semana 4** вҖ” Deploy producciГіn + cliente piloto + bugfix.
- **Fase 2 (1-2 semanas mГЎs)**: integraciГіn Holded (facturas read-only) + n8n para automatizaciones (albarГЎn recibido en `info@`, recordatorios, etc.).

### Lo que NO entra en MVP
- Holded (facturas): fase 2.
- n8n (automatizaciones de email/whatsapp al detectar eventos): fase 2. Mientras tanto el cГіdigo manda emails directamente desde Edge Functions usando Postmark + WhatsApp Cloud API.
- Pasarela de pago: fase 3 (solo si llegamos a aceptar pagos en el panel).
- Roles transportista / proveedor: fase 4 (cuando se valide la operativa con clientes).
